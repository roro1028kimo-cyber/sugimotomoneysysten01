# 會計核銷系統 - 架構設計文件

## 一、系統目標

建立一個直覺、易用的會計核銷系統，讓使用者能夠：
1. 有效管理廠商資訊（公司名稱、電話、銀行帳戶、負責人）
2. 建立、追蹤、核銷會計憑證
3. 快速查詢與篩選資料

## 二、核心設計原則

### 2.1 使用者體驗優先
- **一目了然**：關鍵資訊在首頁即可掌握
- **最少點擊**：常用功能不超過 2 次點擊可達
- **即時回饋**：操作後立即顯示結果

### 2.2 資料架構簡潔
- 廠商與憑證為一對多關係
- 憑證狀態清晰（待核銷、已核銷、已作廢）

## 三、功能模組

### 3.1 廠商管理模組
| 欄位 | 說明 | 必填 |
|------|------|------|
| 公司名稱 | 廠商全名 | ✓ |
| 電話 | 聯絡電話 | ✓ |
| 銀行帳戶 | 匯款用帳戶 | ✓ |
| 負責人姓名 | 主要聯絡人 | ✓ |

功能：
- 新增廠商
- 編輯廠商資訊
- 刪除廠商（需檢查是否有關聯憑證）
- 搜尋/篩選廠商

### 3.2 會計憑證模組
| 欄位 | 說明 | 必填 |
|------|------|------|
| 憑證編號 | 系統自動產生 | 自動 |
| 廠商 | 關聯廠商 | ✓ |
| 金額 | 核銷金額 | ✓ |
| 日期 | 憑證日期 | ✓ |
| 摘要 | 費用說明 | ✓ |
| 狀態 | 待核銷/已核銷/已作廢 | 自動 |
| 建立時間 | 系統記錄 | 自動 |

功能：
- 新增憑證
- 編輯憑證
- 核銷憑證
- 作廢憑證
- 搜尋/篩選憑證

## 四、頁面結構

```
首頁（Dashboard）
├── 統計摘要卡片
│   ├── 待核銷筆數/金額
│   ├── 本月已核銷金額
│   └── 廠商總數
├── 快速操作按鈕
└── 最近憑證列表

廠商管理
├── 廠商列表（含搜尋）
├── 新增/編輯廠商表單
└── 廠商詳情（含關聯憑證）

會計憑證
├── 憑證列表（含篩選）
├── 新增/編輯憑證表單
└── 憑證詳情
```

## 五、UI 設計規範

### 5.1 色彩系統
- 主色：藍色系（專業、信任）
- 成功：綠色（已核銷）
- 警告：橙色（待核銷）
- 危險：紅色（已作廢）

### 5.2 互動設計
- 表格支援排序、分頁
- 表單即時驗證
- 刪除操作需二次確認
- 載入狀態顯示

## 六、技術選型

採用前端靜態應用架構：
- React + TypeScript（類型安全）
- TailwindCSS（快速樣式開發）
- LocalStorage（本地資料持久化）
- 元件化設計（可維護性）
